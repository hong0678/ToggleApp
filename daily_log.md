# 데일리 로그

## 2026-05-06

### 모바일 지도 화면 개선
- `mobile/app/views/map_around.tsx` 지도 화면을 WebView 기반 카카오맵 구조로 유지하면서 터치 레이어 문제를 정리했다.
- 하단 주변 추천 장소 시트가 지도 터치를 막지 않도록 absolute overlay 구조를 단순화했다.
- 현재 위치 자동 이동 로직이 지도 드래그 후 다시 현재 위치로 돌아오게 만들던 문제를 수정했다.
- 현재 위치 버튼을 누르면 에뮬레이터/기기 위치 기준으로 다시 이동하도록 연결했다.

### 카카오 장소 카테고리 검색 연결
- 기존 수동 문자열 카테고리를 `{ label, code }` 구조로 변경했다.
- 카카오 Places 카테고리 코드(`FD6`, `CE7`, `CS2`, `MT1`, `PM9`, `HP8` 등)를 앱 카테고리와 매핑했다.
- WebView 안에서 `kakao.maps.services.Places()`를 사용하도록 `libraries=services`를 추가했다.
- 카테고리 선택 시 `categorySearch` 또는 `keywordSearch`를 실행하고, 지도 마커와 하단 카드 목록을 갱신하도록 연결했다.
- `현 지도에서 검색` 버튼이 현재 지도 중심 기준으로 다시 검색하도록 연결했다.

### 주변 추천 장소 카드 문구 정리
- 카카오 검색 결과 중 우리 서비스에 등록되지 않은 장소는 `상태정보 없음`으로 표시하도록 변경했다.
- 미등록 장소는 거리 옆에 `미등록 장소` 문구를 표시해서, 점주가 상태를 직접 관리하는 등록 매장과 구분되도록 했다.

### 위치 권한 설정

- `mobile/app.json`에 iOS 위치 권한 문구와 Android 위치 권한을 추가했다.
