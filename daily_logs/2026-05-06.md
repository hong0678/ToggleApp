# 2026-05-06 작업 기록

## 1. 오늘 한 작업
- `mobile/app/views/map_around.tsx`의 카카오맵 WebView 기반 지도 화면을 점검했다.
- 지도 위에 올라간 검색바, 카테고리 버튼, 현재 위치 버튼, 하단 주변 추천 장소 시트의 터치 이벤트 구조를 확인했다.
- `map_search.tsx`의 정상 동작 구조와 `map_around.tsx`의 overlay 구조를 비교했다.
- 현재 위치 버튼, 하단 시트 접기/펼치기, 카테고리 필터, 현 지도에서 검색 동작을 함께 정리했다.

## 2. 발생한 문제
- `map_around.tsx`에서 지도 영역이 보이지만 손가락으로 드래그하거나 확대/축소할 때 지도가 자연스럽게 움직이지 않았다.
- 한 손 드래그는 거의 반응하지 않고, 두 손 드래그는 잠깐 반응했다가 다시 원래 위치로 돌아가는 현상이 있었다.
- 주변 추천 장소 하단 시트를 수정한 뒤부터 지도 조작이 더 불안정해졌다.
- 현재 위치가 아닌 다른 지도 영역으로 이동하려고 해도 다시 현재 위치로 돌아오는 것처럼 보였다.

## 3. 원인 분석
- 처음에는 WebView 위에 올라간 React Native absolute overlay가 지도 터치 이벤트를 막는 것으로 의심했다.
- `map_search.tsx`는 WebView가 전체 화면을 차지하고, 하단 시트만 실제 보이는 높이만큼 absolute로 올라가 있었다.
- 반면 `map_around.tsx`는 하단 시트 래퍼와 상단 overlay가 더 복잡했고, 투명 영역이 WebView 위에서 터치를 가로챌 가능성이 있었다.
- 추가로 확인한 결과, 현재 위치를 가져오는 `useEffect`와 `moveToCurrentLocation` 의존성 때문에 위치 상태가 바뀔 때마다 지도가 다시 현재 위치로 이동하는 루프가 있었다.
- 그래서 지도 드래그가 막힌 것처럼 보였지만, 실제로는 드래그 후 `map.setCenter()`가 다시 현재 위치를 지정하는 문제도 함께 있었다.

## 4. 해결한 내용
- WebView 위의 불필요한 투명 wrapper를 제거하고, 하단 시트 자체만 실제 터치 영역이 되도록 구조를 단순화했다.
- 상단 overlay의 고정 높이와 높은 elevation 값을 제거해 지도 위 빈 영역 터치가 WebView로 내려가도록 정리했다.
- WebView 지도 영역을 `StyleSheet.absoluteFillObject` 기반으로 전체 화면에 깔아 `map_search.tsx`와 유사한 구조로 맞췄다.
- 현재 위치 값은 `currentCoordsRef`에 저장하도록 바꿔, `useEffect`가 반복 실행되며 지도를 현재 위치로 되돌리는 문제를 끊었다.
- 현재 위치 버튼을 눌렀을 때만 위치를 다시 가져와 지도 중심을 이동하도록 정리했다.
- 카카오맵 HTML 내부의 지도 옵션에서 `draggable`, `scrollwheel`, `setDraggable(true)`, `setZoomable(true)` 설정을 유지했다.

## 5. 다음 작업
- 실제 기기와 Android Emulator에서 지도 드래그, 핀치 줌, 현재 위치 이동 동작을 반복 확인한다.
- 하단 시트 높이를 조절할 때 지도 터치 영역이 다시 가려지지 않는지 확인한다.
- 카테고리 검색 결과와 하단 카드 목록이 지도 중심 기준으로 자연스럽게 갱신되는지 점검한다.
- 우리 서비스에 등록된 매장과 카카오 검색만으로 표시되는 미등록 장소의 상태 문구를 더 명확하게 구분한다.
